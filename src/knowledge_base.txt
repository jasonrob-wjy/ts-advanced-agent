LangChain 是一个用于开发由语言模型驱动的应用程序的框架。
它使应用程序能够：
- 具备感知能力：将语言模型连接到上下文来源（提示指令、少样本示例、用于支撑回答的内容等）。
- 进行推理：依靠语言模型进行推理（如何根据提供的上下文回答、采取什么行动等）。

LangGraph 是一个用于构建有状态、多角色 LLM 应用程序的库，用于创建 Agent 和多 Agent 工作流。
与其他 LLM 框架相比，它提供了这些核心优势：循环、可控性和持久性。
LangGraph 允许你定义涉及循环的流程，这对大多数 Agent 架构至关重要，将其与基于 DAG 的解决方案区分开来。
它让你能够精细控制应用程序的流程和状态；这对于创建可靠的 Agent 至关重要。
它包含内置的持久性，支持高级的人在回路和记忆功能。

LangGraph 中的主要概念包括：
- State (状态)：一个共享的数据结构，代表应用程序的当前快照。
- Nodes (节点)：编码 Agent 逻辑的 Python 函数。它们接收当前 State 作为输入，并输出对该 State 的更新。
- Edges (边)：控制流规则，根据当前 State 决定下一个执行哪个 Node。
